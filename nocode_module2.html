Module 2: No-Code Data Preparation

Preparing AI-ready text and Q&A datasets with OpenRefine, without writing code.

Why Data Preparation Matters for AI

Before an AI model can classify sentiment, answer questions, or summarize documents, the data must be organized into a clear and consistent structure.

In this module, learners use OpenRefine as a no-code environment for inspecting, cleaning, and exporting model-ready datasets.

The goal is not to build a model yet. The goal is to make the dataset clean enough that a model can use it reliably.

OpenRefine Workflow

OpenRefine is useful for showing the full data-preparation workflow visually:

1. Import

Load a CSV file and confirm that columns are parsed correctly.

2. Inspect

Use facets to understand labels, missing values, and data patterns.

3. Clean

Normalize text, trim whitespace, and correct inconsistent values.

4. Export

Save the cleaned dataset as CSV or JSON for later AI workflows.

Task 1: Sentiment Analysis Data Prep

A sentiment dataset usually contains a text column and a sentiment label such as positive, negative, or neutral.

Recommended OpenRefine demo:

  1. Open the text column.
  2. Select Edit cells → Transform.
  3. Apply lowercase and whitespace cleanup using GREL.
value.toLowercase().trim()

This demonstrates how no-code tools can reproduce common NLP preprocessing steps, such as lowercasing and normalization, in a transparent way.

Facets and Clustering

Facets help learners inspect the dataset. For example, applying a text facet to the sentiment column shows how many positive, negative, and neutral examples are present.

Cluster and edit helps learners fix inconsistent values, such as different spellings, capitalization, or accidental spaces.

Facet = Find Problems

Use it to check label distribution, detect missing values, and spot unusual categories.

Cluster = Fix Problems

Use it to merge similar values such as “Positive,” “positive,” and “positve.”

Task 2: Question Answering Data Prep

A question answering dataset usually needs at least three columns: question, context, and answer.

Important: QA cleaning should be careful. If the answer text changes too much, the answer may no longer match the exact span inside the context.

In OpenRefine, learners can still normalize quotation marks, remove extra spaces, and create helper columns for a SQuAD-style format.

value.replace("“", "\"").replace("”", "\"").trim()

This shows why different AI tasks require different preprocessing decisions.

What Learners Should Be Able to Do

  • Import a CSV dataset into OpenRefine.
  • Confirm that columns are parsed correctly before creating a project.
  • Use GREL transformations for basic text normalization.
  • Use facets to inspect label distribution and detect problems.
  • Use clustering to standardize inconsistent values.
  • Export a cleaned dataset for later AI model development.
